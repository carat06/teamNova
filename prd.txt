1. Executive Summary
TrustID is a mobile-first, user-controlled financial identity verification platform that solves three critical problems in Indian regulated financial services: deepfake-powered impersonation, synthetic identity fraud, and redundant re-KYC across institutions.
The platform enables individuals to register once using their mobile number, complete a full identity verification (documents, video KYC, biometric), and receive a cryptographically signed Verifiable Credential stored in their personal wallet. When any regulated entity — bank, NBFC, broker, or insurer — needs to verify the user, TrustID generates a Zero-Knowledge Proof that confirms required claims (age, PAN status, KYC grade, income band) without exposing any raw Personally Identifiable Information.
TrustID is aligned with the RBI Account Aggregator framework, UIDAI's DigiLocker API, and India's financial inclusion mandate — including offline and voice-first access for low-bandwidth users.

2. Problem Statement
2.1 Current State of Financial Identity Verification
India's financial services sector processes over 200 million KYC verifications annually. The current system has three structural failures:
Redundancy: Every bank, broker, and insurer runs its own independent KYC. A user opening accounts at three institutions submits Aadhaar, PAN, and biometrics three separate times.
PII Exposure: Every verification event creates a new copy of sensitive documents at a new institution, multiplying breach surface.
Fraud Vulnerability: The rise of AI-generated deepfakes and synthetic identities has made document-based and video KYC trivially attackable. Existing systems lack real-time deepfake detection.

4. User Flows
4.1 Registration Flow
Registration is a one-time process that establishes a verified identity for the user.
User enters mobile number (+91 format, 10 digits). System validates format and checks for existing registration.
System sends a 6-digit OTP via SMS. User enters OTP within 30-second window. Three wrong attempts lock the session for 10 minutes.
On OTP verification, user proceeds to document verification. Mobile number is stored as SHA-256 hash — never in plaintext.
User uploads or camera-captures Aadhaar and PAN. OCR extracts structured fields. Name cross-match validates consistency across documents.
User completes 3-challenge Video KYC: (i) face straight + name spoken, (ii) head turn right, (iii) document held to camera.
System runs deepfake detection on every video frame. All three challenges must pass.
On full verification, system issues a Verifiable Credential signed with TrustID's issuer key and stores it in the user's device wallet.
4.2 Login Flow
After registration, users authenticate using facial recognition — no password or PIN required.
System accesses front camera via WebRTC getUserMedia.
Face-api.js detects face presence and alignment in real time.
Passive liveness: blink detection (eye aspect ratio < 0.2 for 2 frames).
On confidence > 0.85 across 3 consecutive frames, face hash is computed and compared against stored credential hash.
Successful match sets session state. Failed match after 3 attempts locks account for 15 minutes.
4.3 Identity Sharing Flow (ZK Proof)
This flow is triggered when an external institution requests identity verification from the user.
External party (bank, NBFC, broker) sends a verification request specifying required claims, e.g.: age_above_18, pan_verified, kyc_grade_A, income_above_3L.
TrustID app presents the request to the user showing each claim in plain English. User reviews and may deselect any claim.
User taps 'Share with proof'. App displays expandable explanation of what ZK proof means and confirms zero PII will be sent.
App generates a ZK proof for each selected claim using the stored Verifiable Credential. Each proof is a cryptographic string derived from the credential via snarkjs circuits.
Only the proof bundle is transmitted: boolean result per claim + proof hash + issuer signature. No name, DOB, Aadhaar number, income figure, or document image is included.
Receiving institution verifies: (i) proof math validity, (ii) credential issuer signature, (iii) revocation status. Verification completes in under 5 seconds.
A consent receipt is written to the user's immutable audit log: timestamp, institution ID, claims shared. User can view full history at any time.

6.2 Security
All PII fields hashed with SHA-256 (crypto.subtle) before storage. Raw values never written to localStorage or any remote store.
Verifiable Credential encrypted with AES-256-GCM using a key derived from the user's face hash + device fingerprint.
TrustID issuer private key stored in HSM (Hardware Security Module) for production; mock RSA hex string for prototype.
All API calls over TLS 1.3. Certificate pinning on mobile app.
OTP sessions time-limited to 30 seconds. OTP tokens are single-use and invalidated on first use.
Consent log entries are append-only. No delete operation exposed in UI or API.
6.3 Privacy
Zero raw PII transmitted in the ZK proof sharing flow. Technically enforced, not policy-only.
Mobile number stored only as SHA-256 hash after registration confirmation.
Face biometric stored only as a derived hash, never as a recoverable image.
DigiLocker documents fetched transiently for OCR extraction only; not persisted server-side.
User can permanently delete all data ('Right to erasure') from Settings — triggers credential revocation and local data purge.
6.4 Accessibility & Inclusion
Voice-guided flow in 12 Indian languages via Bhashini API integration (Hindi, Tamil, Telugu, Kannada, Malayalam, Bengali, Marathi, Gujarati, Punjabi, Odia, Assamese, Urdu).
USSD fallback for feature phones: complete KYC registration via *XXXX# menu without internet.
On-device face match using TensorFlow Lite — works at 50 KB/s without uploading photos.
WCAG 2.1 AA compliance: minimum 4.5:1 contrast, 44px touch targets, screen reader labels on all interactive elements.
Large-text mode: all body text scales to minimum 18sp on accessibility font size setting.

10. Glossary
Term
Definition
Verifiable Credential (VC)
A cryptographically signed digital document containing verified identity claims, issued by a trusted party and held by the subject.
Zero-Knowledge Proof (ZKP)
A mathematical proof that a statement is true without revealing any information beyond the truth of the statement itself.
Deepfake
AI-generated synthetic media — typically a face — that mimics a real person's appearance to defeat identity verification systems.
Liveness Detection
Technical measures to distinguish a live person from a photo, video, or mask during biometric capture.
DigiLocker
India's official digital document wallet service operated by MeitY, containing Aadhaar, PAN, driving licence, and other government-issued documents.
Account Aggregator (AA)
RBI-licensed data-sharing framework enabling consented financial data flows between financial information providers and users.
KYC Grade
A risk-based classification (A/B/C) assigned after full identity verification, indicating the level of confidence in the user's identity.
Synthetic Identity
A fraudulent identity created by combining real attributes (e.g., a valid PAN) with fabricated ones (fictitious name, address) to pass document-based checks.
snarkjs
A JavaScript library implementing the zk-SNARK zero-knowledge proof system, usable in browsers via WebAssembly.
BBS+ Signatures
A cryptographic signature scheme enabling selective disclosure — proving a subset of signed attributes without revealing the full set.
USSD
Unstructured Supplementary Service Data — a text-menu protocol on mobile networks, usable without internet on feature phones.
HSM
Hardware Security Module — a physical device that safeguards cryptographic keys and performs signing operations in a tamper-proof environment.
